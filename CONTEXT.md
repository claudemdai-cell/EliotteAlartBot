## PROYECTO A CONSTRUIR — ELLIOTT ALERT BOT

### El problema que resuelve
José analiza bien pero el timing falla porque no está mirando el gráfico en el momento exacto.
El trade ETH (mayo 2026) entró sin las 5 capas porque esperaba manualmente.

### Arquitectura propuesta
```
TradingView (Pine Script)
    ↓  webhook cuando condición se cumple
Python server (Flask/FastAPI)
    ↓  evalúa las 5 capas con datos en tiempo real
    ↓  si ≥ 4/5 capas OK:
Notificación → Telegram Bot
    ↓
José decide si entrar
```

### Las 5 capas a verificar automáticamente
```python
# Capa 1: Precio en zona Elliott (dato del webhook de TradingView)
in_elliott_zone = True  # viene del Pine Script

# Capa 2: Fibonacci 50-61.8%
fib_50 = wave_start - (wave_range * 0.50)
fib_618 = wave_start - (wave_range * 0.618)
in_fib_zone = fib_618 <= current_price <= fib_50

# Capa 3: Volumen decreciente (promedio últimas 5 velas < promedio 20 velas)
volume_decreasing = avg_vol_5 < avg_vol_20

# Capa 4: RSI 4H < 40 (zona oversold, divergencia pendiente)
rsi_ok = rsi_4h < 40

# Capa 5: Vela de reversión (precio cierra arriba del open en 4H)
reversal_candle = close_4h > open_4h

capas_ok = sum([in_elliott_zone, in_fib_zone, volume_decreasing, rsi_ok, reversal_candle])
if capas_ok >= 4:
    send_telegram_alert(asset, capas_ok, current_price, stop_level, target_level)
```

### APIs disponibles
- **CoinDesk MCP**: precios, noticias, OHLCV histórico
- **Crypto.com MCP**: orderbook, tickers en tiempo real
- **TradingView webhooks**: alertas por Pine Script (gratis con cuenta básica)
- **Telegram Bot API**: notificaciones gratis
